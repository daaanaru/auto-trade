## 3. Sharpe改善のボトルネック分析

### 現在のSharpe: -3.234（年率換算、卒業checker方式）

計算方法: paper_portfolio_log.jsonの日次total_value_jpyからpct_changeを算出 → mean/std × √252

| 要素 | 値 |
|------|-----|
| 日次平均リターン | -0.2086% |
| 日次標準偏差 | 1.0239% |
| 営業日数 | 17日 |

### 核心問題: 3/16-19の連続SLクラッシュ

3/16-19に27件中13件のSLが集中発生。ポートフォリオ日次リターンで:
- 3/18: **-2.225%**
- 3/19: **-2.139%**

この2日が標準偏差を1.02%まで押し上げている。**暴落2日を除外するとSharpeは+1.607**に跳ね上がる。

つまり「戦略が悪い」というより「SL集中発生で一気に資産が削られた」のが根本原因。

### Sharpe 0.5達成に必要な条件

残り14日間で毎日一定のリターンを出した場合の試算（既存17日分の実績は固定）:

| 日次リターン | 最終資産 | 総リターン | Sharpe |
|-------------|----------|-----------|--------|
| +0.0% | 289,880円 | -3.4% | **-2.374** |
| +0.2% | 298,103円 | -0.6% | -0.372 |
| **+0.3%** | **302,295円** | **+0.8%** | **+0.585** |
| +0.5% | 310,844円 | +3.6% | +2.365 |

**残り14日で毎日+0.3%（約870円/日）を安定して出せればSharpe 0.5をクリアできる。**

ただし「毎日一定」はあり得ないので、実際には:
- 大きなSLを出さないこと（標準偏差を上げない）
- コツコツ利益を積み上げること（平均をプラスに転換）

の両立が必要。

## 4. SL/TP パラメータ見直しの逆算

### 現行パラメータ

| パラメータ | 値 | 備考 |
|-----------|-----|------|
| SL | ATR×1.5 | キャップ-5%、下限-1.5% |
| TP1 | ATR×3.0 | 下限+3% |
| TP2 | ATR×5.0 | 下限+6% |
| トレーリング | 高値から ATR×1.0 | +3%到達で発動、-1.5%で決済 |
| 強制決済 | 7日 | |

### 過去データからの逆算

**SLの実績**:
- bb_rsi/us SL平均: -762円（SLキャップ-5%近辺で刈られている）
- monthly/jp SL平均: -1,049円（日本株は1ポジション金額が大きいため損失額も大）
- volscale_sma/btc SL平均: -769円

**TP1の実績**:
- bb_rsi TP1平均: +301円
- monthly TP1平均: +170円
- volscale_sma TP1平均: +228円

**問題**: SL平均(-762〜-1,049円) vs TP1平均(+170〜+301円) でRR比が1:3〜1:6で不利。

### 改善シナリオ

#### A. SLキャップを-5%→-3%に縮小

monthly SLの6件に適用した場合: -6,293円 → 推定-3,776円（**+2,517円の改善**）
全SLに適用した場合: -22,324円 → 推定-13,394円（**+8,930円の改善**）

リスク: 低ボラ銘柄で「ノイズによるSL狩り」が増える。ただし現行SL_FLOOR=-1.5%があるので最低限の幅は確保される。

#### B. TP1下限を+3%→+5%に引き上げ

現在TP1は+3%で1/2利確しているが、TRAILING_STOPの平均利益が+390〜+419円とTP1の+170〜+301円を上回る。TP1を遅らせてトレーリングに乗せる方が期待値が高い。

TP1を+5%にした場合、現在のTP1の22件のうち一部はSLに変わるリスクがあるが、通過する銘柄の利益は+3%→+5%に増える。

#### C. 強制決済7日→5日に短縮

FORCE_EXITの6件は平均+202円。方向感がない銘柄を早めに切ることで資金効率を上げる。

## 5. 具体的な提案（3案）

### 案A: 保守的（リスク最小化重視）

**方針**: SLによる大損を防ぎ、stdを下げることでSharpeの分母を改善

| 変更項目 | 現行 | 変更後 |
|---------|------|--------|
| 稼働戦略 | monthly(jp) + vol_div(btc) + volscale_sma(btc: BTC+ETH) | **変更なし** |
| SLキャップ | -5% | **-3%** |
| TP1下限 | +3% | **変更なし** |
| 強制決済 | 7日 | **5日** |
| ポジション枠 | 現状維持 | 現状維持 |

**期待効果**:
- SLキャップ縮小で1回あたりの最大損失が約40%減（-762円→-457円相当）
- 強制決済短縮で塩漬けリスク低減
- **日次std低下が主たるSharpe改善要因**

**リスク**: SL狩りが増える可能性。特にBTC市場のボラティリティでは-3%が浅すぎる可能性あり

**Sharpe到達見込み**: SLが減りstdが抑制されれば0.0〜+0.5の範囲。大きなプラス転換は期待しにくい

---

### 案B: 中間（戦略集中+パラメータ調整）— 推奨

**方針**: 実績ベースで最も優秀なvolscale_sma(jp)に枠を集中。パラメータも調整

| 変更項目 | 現行 | 変更後 |
|---------|------|--------|
| 稼働戦略 | monthly(jp) + vol_div(btc) + volscale_sma(btc: BTC+ETH) | **volscale_sma(jp: 5枠) + vol_div(btc) + volscale_sma(btc: ETHのみ)** |
| monthly | 稼働中 | **新規エントリー停止**（ペイオフ0.21は構造的に不利） |
| volscale_sma(jp) | 2件のみ | **最大5件に拡大** |
| SLキャップ | -5% | **-3.5%**（jp株用。BTCは-5%維持） |
| TP1下限 | +3% | **+4%** |
| 強制決済 | 7日 | **5日** |

**根拠**（全て実データ）:
- volscale_sma(jp): 6件で勝率67%、ペイオフ2.60、Sharpe近似+0.767。**唯一の安定黒字戦略**
- volscale_sma(jp)のSL平均が-157円と極めて小さい（SLキャップに到達する前にVolScale独自出口で切れている可能性）
- monthly(jp): 勝率66.7%だがSL平均-1,049円でペイオフ0.21。勝っても利益が薄く（TP1平均+170円）、1回のSLで6勝分が消える
- vol_div: 2件2勝で+642円。サンプル不足だが現時点でマイナス要素なし

**期待効果**:
- volscale_sma(jp)の実績ペース（日次+331円/稼働日）が枠拡大で加速
- monthly停止でSL-1,049円級の大損を構造的に排除
- **日次+0.3%ラインの現実的な達成可能性**: volscale_sma(jp) 5枠 × 日次+66円/枠 = +330円。残り14日で+330×14 = +4,620円。最終約294,500円（リターン-1.8%）。追加でvol_divとETHが寄与すれば+0.3%/日に接近

**Sharpe到達見込み**: +0.3〜+1.0。SL集中が再発しなければ0.5超えの可能性あり

**リスク**: volscale_sma(jp)のサンプル6件は統計的に脆弱。好成績が偶然の可能性は否定できない

### 案C: 攻め（大幅変更+利益最大化）

**方針**: 損失戦略を全停止、黒字戦略に全振り。パラメータも攻めに調整

| 変更項目 | 現行 | 変更後 |
|---------|------|--------|
| 稼働戦略 | monthly(jp) + vol_div(btc) + volscale_sma(btc: BTC+ETH) | **volscale_sma(jp: 8枠) + vol_div(btc: 3枠)** |
| monthly | 稼働中 | **新規エントリー停止** |
| volscale_sma(btc) | BTC+ETH | **停止**（btcでのペイオフ0.66は不利） |
| volscale_sma(jp) | 2件 | **最大8件に拡大** |
| vol_div(btc) | 1件 | **最大3件に拡大** |
| SLキャップ | -5% | **-3%** |
| TP1下限 | +3% | **+5%** |
| 早期TS発動 | +3% | **+4%** |
| 強制決済 | 7日 | **4日** |
| 1銘柄上限 | 4% | **6%**（枠集中に合わせて拡大） |

**根拠**:
- volscale_sma(btc)の26件は勝率54%/ペイオフ0.66で期待値マイナス。日本株に集中すべき
- vol_divは2勝0敗+642円。MONA shortが稼働中で方向性は良い。枠拡大で勝ちパターンを複製
- TP1を+5%に引き上げ: 現在+3%で半分利確→そのまま+5%まで伸ばせば利益1.67倍。volscale_sma(jp)のTP1平均+310円が+517円に改善する可能性
- 1銘柄上限6%: 少数精鋭で1件あたりの利益額を増やす

**期待効果**:
- volscale_sma(jp)のペイオフ2.60を最大活用
- SL縮小+btcボラ回避で日次stdを0.7%以下に抑制
- **日次+0.5%達成時のSharpe: +2.365**

**Sharpe到達見込み**: +0.5〜+2.0。ただしvolscale_sma(jp)のサンプル6件に全てを賭ける形

**リスク**:
- **最大のリスク: サンプル6件への過学習**。volscale_sma(jp)の好成績が続く保証はない
- 1銘柄上限6%はSL時の損失額も1.5倍になる
- jp市場は平日のみ稼働。土日は利益が出ない（vol_divのbtcがカバー）

## 6. 総合所見

### Sharpe 0.5到達は「不可能ではないが厳しい」

- 必要条件: 残り14日で毎日+0.3%（≒870円/日）の安定リターン
- 現実: 最良戦略volscale_sma(jp)ですら日次+331円。枠拡大しても500円/日程度が上限
- **SLを1回出さないことが最重要**。SL1回(-769円)で2日分の利益が消える

### 最大のレバー: 「SLを出さない」＝「SLキャップ縮小」+「ボラが高い市場を避ける」

| 施策 | Sharpe影響（分子改善） | Sharpe影響（分母改善） |
|------|----------------------|----------------------|
| SLキャップ縮小 | ○（損失額減少） | ◎（大暴落日の排除） |
| monthly停止 | ○（SL-1,049円排除） | ○（ドローダウン日削減） |
| volscale(btc)停止 | △（小幅改善） | ○（BTCボラ由来のstd削減） |
| volscale(jp)拡大 | ◎（黒字戦略の増幅） | △（利益日のstdは上げる） |
| TP1引上げ | ○（1件あたり利益増） | △ |

### 推奨: 案B（中間案）を即時実行

理由:
1. volscale_sma(jp)は唯一ペイオフレシオが2を超える戦略。リスクリワードが構造的に優位
2. monthly停止は「勝率が高くても儲からない」問題の根本解決
3. SLキャップ-3.5%は-5%と-3%の中間で、SL狩りリスクと損失抑制のバランスが取れる
4. 残り14日という短期間では、サンプル数を増やすより「負けない」方が重要
