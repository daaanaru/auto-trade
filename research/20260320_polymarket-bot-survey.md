## 1. Polymarketとは何か

**Polymarket = 予測市場（Prediction Market）プラットフォーム**

- Polygon（Ethereumの高速版チェーン）上で動く分散型の「賭け市場」
- 「ある出来事が起こるか・起こらないか」に対してYES/NOのトークンを売買する
- 価格は0〜$1で、市場参加者の合意による確率を反映する（YES=$0.70 = 70%の確率で起こると市場が判断）
- 結果が確定すると、正解側のトークンが$1に、不正解側が$0になる
- **CLOB（Central Limit Order Book）方式**: 従来のAMM（自動値付け）ではなく、取引所のような指値注文帳を使用。これが高頻度botの参入を可能にしている

### 特徴

- 2024年米大統領選で爆発的に成長（累計取引高$10B超）
- 政治・スポーツだけでなく、**BTC/ETH価格の短期予測マーケット**が活発
- UIはWeb + モバイルアプリ。API/SDKも公開されており、bot参入の障壁が低い

## 2. BTC価格予測マーケットの仕組み

### 5分/15分マーケット

- **「5分後（または15分後）にBTC価格は上がるか下がるか」** というバイナリ市場
- 5分ごとに新しいマーケットが立ち上がる（連続開催）
- YES（上がる）とNO（下がる）のトークンを売買
- **YES + NOの合計 = $1**が理論値だが、スプレッドにより$1を下回ることがある
- 満期（5分/15分後）に結果が確定し、正解側が$1を受け取る

### 収益構造

- 方向性賭け: BTC価格が上がると予測→YES購入→的中すれば$1回収
- アービトラージ: YES + NO < $1のタイミングで両方購入→確実に利益（スプレッド分）
- 速度優位: 実価格の変動をPolymarketより早く検知して先行エントリー

## 3. @noisyb0y1が言及した「実価格を10-15ms早くキャッチ」の手法

### 仕組み（Noisy氏の主張）

```
実際のBTC価格変動     → t = 0ms
Botが価格変動を検知    → t = +10-15ms
Polymarket価格に反映   → t = +20-35ms
                       ──────────
                       Δt ≈ 5ms の利用可能ウィンドウ
```

- **BinanceやBybitなどの取引所WebSocketから実価格をリアルタイム取得**
- Polymarketのオッズ更新は数十ms〜数秒遅れる
- その「遅延ウィンドウ」を利用して先行エントリーする
- Noisy氏の引用事例: 0x8dxd氏が$313→$729,189（1ヶ月）、k9Q2mX4L8A7ZP3R氏が$1,277→$510,000

### 技術的要件

- 取引所WebSocket接続（BinanceまたはBybitの1秒足/ティックデータ）
- Polymarket CLOB APIへの低遅延接続
- 価格乖離検知 → 即時注文のパイプライン
- **サーバーの地理的位置が重要**: Polygon RPCノードに近いデータセンターほど有利

### 信頼性評価【重要】

| 項目 | 評価 |
|------|------|
| 手法の概念 | **実在する**。取引所間の価格遅延アービトラージは古典的手法 |
| Noisy氏の事例数値 | **誇張の疑い大**。$313→$729,189は現実的でない |
| OpenClaw/SpartanLabs への誘導 | **スキャム的要素あり**。アフィリエイトリンク付き |
| 5msウィンドウの実用性 | **極めて厳しい**。機関投資家レベルのインフラが必要 |
| DaNARU社内の過去評価 | OpenClaw調査レポートで「疑わしい」「参考のみ」と判定済み |

## 4. ボットで参加する方法（API/SDK）

### 公式API

- **Polymarket CLOB API**: RESTful + WebSocket
  - 注文の作成・キャンセル・照会
  - マーケット情報の取得
  - リアルタイム価格フィード
- **Polymarket CLOB Client（Python SDK）**: `pip install py-clob-client`
  - GitHub: Polymarket公式リポジトリで公開
  - Python 3.8+対応

### 認証・ウォレット

- Polygon上のEthereumウォレット（MetaMaskなど）が必要
- APIキーはPolymarketアカウントから取得
- 署名にはプライベートキーが必要（EIP-712署名）

### GitHub上の参考リポジトリ（DaNARU巡回で発見済み）

- **apeethowl0x/polymarket-trader-bot**（71 stars）: クジラウォレット追跡+CLOB自動執行。ただしコピートレード系なので過度な信頼は禁物
- その他、polymarket-apitrader, polymarket-hedgebot, polymarket-neutral-bot等のトピックで多数のリポジトリが存在

### 最小構成（もしやるなら）

```
1. Polygonウォレット作成（MetaMask）
2. USDCをPolygonネットワークに入金
3. Polymarketアカウント作成+API key取得
4. py-clob-clientでbot実装
5. BinanceのWebSocketで実価格監視
6. 価格乖離検知→CLOB APIで発注
```

## 5. 日本からの利用可否（法規制）

### 結論: **グレーゾーン（高リスク）**

| 観点 | 状況 |
|------|------|
| Polymarket側の制限 | **米国居住者は利用禁止**（CFTC和解後）。日本は明示的にブロックされていないが、VPN使用を前提としている地域もある |
| 日本の賭博法 | 海外サーバーの「賭博」に日本から参加する行為は、刑法第185条（賭博罪）に抵触する可能性がある |
| 金融商品取引法 | 予測市場のトークンが「デリバティブ」に該当する場合、無登録の取引所での取引は違法になりうる |
| 暗号資産規制 | PolymarketはPolygon上のUSDCで取引。暗号資産交換業の規制（資金決済法）との関係が不明確 |
| 実務上の問題 | KYC（本人確認）なしで参加可能な場合があるが、将来的に規制強化で資金凍結のリスク |
| 税務 | 仮に利益が出た場合、雑所得（最大55%課税）。海外プラットフォームからの出金記録が税務申告上の課題 |

### 過去のDaNARU社内評価

- 2026-03-17レポート: 「予測市場操作リスク顕在化。シグナルソース採用は見送り」
- HackerNews記事（984pt）: Polymarketの賭け参加者が記者を殺害脅迫した事件 → 市場操作リスクの証拠

## 6. 必要資金・手数料

| 項目 | 金額 |
|------|------|
| 最小入金額 | 制限なし（ただし手数料を考慮すると$100〜が現実的） |
| 取引手数料 | Maker: 0%（指値注文）、Taker: 約1-2%（成行注文） |
| ガス代（Polygon） | 極めて安い（1トランザクション$0.01以下） |
| 入金経路 | CEX（Binanceなど）→ Polygon USDC → Polymarket。またはクレジットカード直接入金（地域による） |
| 推奨テスト資金 | $50-100（小額で動作確認） |
| 本格運用 | $1,000+（Noisy氏は少額からスケールしたと主張するが根拠不明） |

## 7. DaNARU社での実現可能性（Mac mini M4環境）

### 技術的には可能

| 要件 | DaNARU環境 | 判定 |
|------|-----------|------|
| Python実行環境 | Mac mini M4 16GB | OK |
| WebSocket常時接続 | launchdで管理可能 | OK |
| 低遅延通信 | **日本→US東海岸は物理的に100ms以上** | NG |
| Polygonノード | パブリックRPCは遅延が大きい。専用ノードは月$50-200 | 条件付き |
| 24時間稼働 | Mac mini常時稼働中 | OK |
| USDC入金 | Bybit経由で可能（bitFlyer→Bybit→Polygon） | OK |

### 致命的問題: **レイテンシ**

- Polymarketのサーバーは米国（おそらくAWS us-east-1）
- 日本からのラウンドトリップ: **100-200ms**
- 5msウィンドウを狙う手法は**日本からは物理的に不可能**
- VPS（AWSやGCPのus-east-1リージョン）を借りれば解決するが、月$20-50の追加コスト

## 8. リスク評価

### 法的リスク 【高】

- 日本の賭博法・金融商品取引法との抵触可能性
- 規制環境が流動的（2025年後半からG7で予測市場規制の議論が活発化）
- 最悪ケース: 資金凍結+法的措置

### 技術的リスク 【中〜高】

- 5msウィンドウは機関投資家のHFT（超高速取引）と同じ土俵。個人が勝てる構造ではない
- Polymarket側が遅延を修正すれば手法自体が死ぬ（@0x_Discoverの「500ms遅延削除後も稼働」は疑わしい）
- スマートコントラクトのバグ・ハッキングリスク

### 資金的リスク 【中】

- 小額テスト（$50-100）なら損失限定
- ただしアービトラージは薄利多売。手数料・ガス代・スリッページで実質赤字になりやすい
- VPS費用やRPCノード費用が固定コストとして積み上がる

### 評判リスク 【低〜中】

- Polymarket周辺はスキャム・誇大広告が多い。関与自体がブランド毀損のリスク
- note記事やX投稿で「やってみた」系のコンテンツにすれば逆にプラスにも

## まとめ: DaNARU社への推奨

| 項目 | 判定 | 理由 |
|------|------|------|
| 即時参入 | **非推奨** | 法的リスクが不透明。レイテンシで勝てない |
| 手法の学習・検証 | **推奨** | 予測市場×botの概念理解はauto-tradeに有用 |
| 小額テスト（$50） | **条件付き可** | 法的リスクを上様が許容するなら。VPSは必要 |
| コンテンツ化（note/X） | **推奨** | 調査結果自体が記事ネタになる |
| シグナルソースとして利用 | **見送り** | 市場操作リスク（脅迫事件の先例）で信頼性が低い |

### 既存auto-tradeとの比較

| | auto-trade（現行） | Polymarket bot |
|---|---|---|
| 法的安全性 | 高（国内証券/取引所） | 低（グレーゾーン） |
| 技術難易度 | 中 | 高（HFTレベル） |
| 必要インフラ | Mac mini単体 | Mac mini + US VPS |
| 期待利回り | 検証中（卒業判定4/12） | 不明（誇大広告が多い） |
| スケーラビリティ | 高（多市場対応済み） | 低（BTC短期のみ） |

## 次に掘るべき論点

**Polymarketの「方向性賭け」ではなく「スプレッドアービトラージ（YES+NO<$1の裁定取引）」だけに絞った場合の期待値を、過去のマーケットデータから定量的に検証する価値がある**（@0x_Discoverの手法の本質はこちら）。
