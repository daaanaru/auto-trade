## 3. 3/16攻め化パラメータの効果検証

### 3.1 前後比較

| 期間 | 件数 | 勝率 | PnL | RR比 |
|------|------|------|-----|------|
| 改善前（〜3/16昼） | 11件 | 64% | -2,634円 | 0.16 |
| 改善後（3/16昼〜） | 56件 | 48% | -5,336円 | 0.56 |
| 改善後（FOMC除外） | 48件 | 56% | **+1,342円** | **0.94** |

### 3.2 判定

**採用案**: パラメータ改善は有効。RR比 0.16 → 0.94（FOMC除外時）と劇的改善。

- SLキャップ -8% → -5%: 損切額が概ね750円前後に収束。旧設定なら-2,000円超の損切が複数発生していた（QDレーザ-2,226円、サムコ-1,336円は改善前の遺物）
- TP1 3.0x / 下限+3%: TP1平均利益+171円、TS平均+207円、TP2平均+400円

**却下案**: 旧パラメータ（SL -8%, TP1 2.0x）への回帰。RR比0.16では期待値がマイナスのまま。

## 4. TP1下限+5%引上げの検討

### 4.1 シミュレーション結果

- 現行TP1（下限+3%）で利確した13件のうち、**12件が+5%未満で発動**
- +5%に引き上げた場合の追加利益: **+41円**（ほぼ中立）

### 4.2 判定

**却下（保留を推奨）**。

理由:
- TP1抑制で保持した分がTP2/TSまで到達するケースと、利益が減るケースが拮抗
- XRP(+316円)やETH(+146円)では有利だが、QDレーザ(-204円)やWFC(-111円)では逆効果
- **サンプル数13件では統計的に結論を出せない。2週間追加データ蓄積後に再検討**

**代替案**: TP1の売却比率を1/2 → 1/3に変更（利確しつつ残存ポジションを増やす折衷案）。これもデータ不足のため3/23時点では実行せず、4月第1週の次回レビューで判断。

## 5. FOMC期間除外フィルター

### 5.1 被害の定量化

| 項目 | 値 |
|------|------|
| FOMC期間損切 | 8件 |
| 損失額 | -6,678円 |
| 全損失に占める割合 | 47% |
| 3/18エントリー分 | 7件 -5,937円 |

**全損失の約半分がFOMC1イベントに集中**。

### 5.2 設計案

```
event_calendar.json に重要イベントを登録:
  - FOMC（年8回）
  - 雇用統計（毎月第1金曜）
  - CPI（毎月）
  - PCEデフレーター（毎月）

unified_paper_trade.py のエントリー判定に:
  「次の重要イベントまで24時間以内 → 新規エントリー禁止」
  「既存ポジションの損切/利確は通常通り稼働」
```

### 5.3 判定

**採用（3/23レビュー後に実装推奨）**。

- 実装コスト: 低（event_calendar.jsonは既存、エントリー関数に1条件追加）
- 期待効果: -5,937円/イベント回避。年8回FOMCだけでも-47,000円のリスク軽減
- 副作用: イベント後の急騰に乗り遅れるリスクあり。ただし損失回避＞機会損失

**トレードオフ**: イベント直後の反発に乗れない。対策として「イベント後12時間はエントリー解禁」とする。
